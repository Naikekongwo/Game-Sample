# OCMP 地图编辑器

> 刘骐源 二零二六年五月十六日

基于 Python Tkinter 的 `.ocmp` 地图文件可视化编辑工具。

---

## 一、文件格式

`.ocmp` 是 OpenCore 引擎的地图二进制格式，由 `MapHeader` + `BlockInfo[]` 组成。

### 1.1 MapHeader（36 bytes，MSVC 默认对齐）

| 偏移 | 大小 | 字段 | 说明 |
|------|------|------|------|
| 0 | 4 | `magic` | 魔数 `"OCMP"` |
| 4 | 1 | `version` | 地图格式版本 |
| 5 | 1 | *(padding)* | 对齐填充 |
| 6 | 2 | `width` | 地图宽度（Block 数），LE |
| 8 | 2 | `height` | 地图高度（Block 数），LE |
| 10 | 1 | `layerCount` | 地图层级数 |
| 11 | 1 | `blockSize` | 每 Block 存储字节数（通常为 4） |
| 12 | 24 | `reserved` | 保留字段 |

### 1.2 BlockInfo（blockSize bytes，通常 4）

| 偏移 | 字段 | 默认值 | 说明 |
|------|------|--------|------|
| 0 | `Terrain` | 2 | 地形纹理索引 |
| 1 | `STRuct` | 2 | 结构/装饰纹理索引 |
| 2 | `Entity` | 2 | 预置实体标记 |
| 3 | `Access` | 2 | 通行性：0=阻挡，非0=可通行 |

## 二、启动方式

### 命令行

```
python tools/ocmp_map_editor.py [地图文件.ocmp]
```

可选传入 `.ocmp` 文件路径，启动后自动加载。

### 批处理

双击 `tools/run_editor.bat`（需根据本机 Python 路径调整脚本中的 `PYTHON` 变量）。

### 依赖

- Python 3.x
- Tkinter（Python 标准库自带）

## 四、功能说明

### 4.1 文件操作

| 操作 | 快捷键 | 说明 |
|------|--------|------|
| 新建 | `Ctrl+N` | 创建指定宽高的空白地图 |
| 打开 | `Ctrl+O` | 加载 `.ocmp` 文件，自动检测 header 对齐方式 |
| 保存 | `Ctrl+S` | 覆盖保存至原文件 |
| 另存为 | — | 选择新路径保存 |

新建时可指定 `blockSize`（默认为 4）。若 blockSize > 4，超出 BlockInfo 结构体大小的字节自动填充默认值。

### 4.2 显示模式

| 模式 | 说明 |
|------|------|
| **综合** | Terrain 为底色，STRuct 右上角标、Entity 左下角标，Access 边框（红=阻挡/绿=通行） |
| Terrain | 仅显示地形纹理索引 |
| STRuct | 仅显示结构纹理索引 |
| Entity | 仅显示实体标记 |
| Access | 仅显示通行性 |

### 4.3 绘图工具

| 工具 | 操作 | 说明 |
|------|------|------|
| **画笔** | 左键拖拽 | 将当前位置的格子设为画笔预设值 |
| **取样器** | 左键点击 | 从点击的格子提取值到画笔 |
| **填充** | 左键点击 | 连通区域批量替换（替换所有与目标值相同的相邻格） |

### 4.4 画笔预设

| 预设 | Terrain | STRuct | Entity | Access | 用途 |
|------|---------|--------|--------|--------|------|
| 可通行地板 | 3 | 0 | 2 | 1 | 角色可行走区域 |
| 墙壁/阻挡 | 4 | 0 | 2 | 0 | 不可通行的障碍 |
| 水/障碍 | 6 | 0 | 2 | 0 | 自然障碍 |
| 空地(默认) | 2 | 2 | 2 | 2 | 默认值填充 |

### 4.5 属性面板

选中格子后可直接修改四个字段的 Spinbox 数值（0~255），点击"应用"按钮生效。也可右键取色同步画笔。

### 4.6 视图操作

| 操作 | 方式 |
|------|------|
| 放大 | 滚轮上滚 / `+` 键 / `Ctrl++` |
| 缩小 | 滚轮下滚 / `-` 键 / `Ctrl+-` |
| 平移 | 鼠标中键拖拽 |
| 重置视图 | 菜单 → 视图 → 重置视图 |
| 清空地图 | 菜单 → 视图 → 清空地图（所有格填充默认值） |

## 五、编辑器与引擎的对应关系

```
编辑器                          OpenCore 引擎
──────                          ────────────
新建/保存 .ocmp    ──────────→  MapLoader::LoadMapFromFile()
                                 ↓
画笔设置 Access     ──────────→  BlockInfo::Access (0=阻挡)
                                 ↓
                               Entity::canMoveTo() 读取并判断碰撞
```

- 编辑器中 Access 字段的修改直接影响游戏中实体能否通过该格
- Terrain/STRuct 对应 `Tile::Draw()` 中使用的纹理索引（`bInfo->Terrain`）
- Entity 字段预留用于标记该格对应的预置实体类型

---

## 六、注意事项

1. 编辑器读写的是 MapLoader.hpp 定义的二进制格式，与引擎完全兼容
2. 大尺寸地图初次加载时可能较慢（需解析所有 Block），建议地图尺寸控制在 512×512 以内
3. 保存时以 `blockSize=4` 的 aligned header 格式写入
4. 旧版地图文件如果 header 对齐方式不同，编辑器会自动检测兼容加载
